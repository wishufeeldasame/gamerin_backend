## 2. 완료된 작업 세부 내역
### Step 7 (NEW)
- **멘토링 신청**: 멘티가 특정 프로그램에 신청 시 마일리지 차감 및 에스크로 결제 상태(`ESCROW_HELD`)
설정.
- **신청 내역 조회**: 멘티(본인이 신청한 내역)와 멘토(본인에게 들어온 신청 내역) 각각에 대한 페이징
조회 API 구현.
- **수락 및 거절**:
  - **수락**: 신청 상태를 `ACCEPTED`로 변경.
  - **거절**: 신청 상태를 `REJECTED`, 결제 상태를 `REFUNDED`로 변경하고 멘티에게 마일리지를 즉시
환불(`addBalance`).
- **데이터 정합성**: `@Transactional` 및 도메인 모델 내 비즈니스 로직(deduct, addBalance)을 통한
안전한 마일리지 처리.
- **보안/권한**: 본인의 프로그램에 신청 불가, 본인에게 온 신청만 수락/거절 가능하도록 검증 로직 추가

- 현재 구현 상 SQL Shell 을 통해 직접 mileage_wallet 테이블에 balance 를 수동으로 채워줘야함.

### 이전 작업 (Step 1 ~ 6)
- 멘토 프로필 관리 및 프로그램(상품) CRUD 로직 완료.
- 게임별 필터링 및 페이징을 포함한 목록 조회 기능 구현.

## 3. 기술적 해결 사항 (Troubleshooting)
- **마일리지 환불 로직**: `MileageWallet` 엔티티에 `addBalance` 메서드를 추가하여 거절 시의 환불
프로세스를 도메인 주도적으로 구현.
- **상태 전이 검증**: `APPLIED` 상태인 경우에만 수락/거절이 가능하도록 상태 체크 로직을 구현하여
비즈니스 규칙 준수.

## 4. 다음 작업 예약 (Step 8)
- **멘토링 완료**: 수업 종료 후 멘티가 완료 확정을 하면 마일리지가 멘토에게 최종 지급(`SETTLED`)되는
로직.
- **리뷰 시스템**: 완료된 멘토링에 대해 평점과 후기를 남기는 기능.
- **통계 업데이트**: 리뷰 평점이 멘토 프로필의 평균 평점에 반영되도록 구현.
